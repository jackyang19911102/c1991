## 一、代码安全

### 🔴 高风险

#### 1.1 第三方 CDN 脚本缺少 SRI 完整性校验

**涉及文件：**
- `blog.html:11` — marked.js 从 `cdn.jsdelivr.net` 加载
- `index.html:102` — 不蒜子统计从 `busuanzi.ibruce.info` 加载

**问题描述：** 两个外部脚本均未使用 `integrity` 属性进行子资源完整性校验（SRI）。如果 CDN 被入侵或中间人攻击，攻击者可注入恶意代码，窃取用户信息或篡改页面内容。

**当前代码：**
```html
<!-- blog.html:11 -->
<script src="https://cdn.jsdelivr.net/npm/marked/marked.min.js"></script>

<!-- index.html:102 -->
<script async src="https://busuanzi.ibruce.info/busuanzi/2.3/busuanzi.pure.mini.js"></script>
```

**修复建议：**
```html
<!-- 方案一：添加 SRI hash -->
<script src="https://cdn.jsdelivr.net/npm/marked/marked.min.js"
        integrity="sha256-XXXX..."
        crossorigin="anonymous"></script>

<!-- 方案二（推荐）：将关键库本地化部署，避免外部依赖 -->
<script src="/js/marked.min.js"></script>
```

**影响范围：** 所有访问 blog.html 和不蒜子统计功能的用户。

---

#### 1.2 不蒜子统计服务依赖外部不可控域名

**涉及文件：** `index.html:102`

**问题描述：** `busuanzi.ibruce.info` 为第三方个人维护的统计服务，存在以下风险：
- 服务可用性不可控（可能随时停止服务）
- 数据发送到第三方服务器，存在隐私合规风险
- 该域名的 JavaScript 可读取当前页面的所有 DOM 信息

**修复建议：**
- 方案一：切换至自建统计（如 Plausible、Umami 自部署版）
- 方案二：使用 Google Analytics 等更稳定的商业统计服务
- 方案三：完全移除访问统计，保持"无追踪"理念与 about.html 中"没有广告"的宣言一致

## 二、代码BUG

### 🔴 高风险

#### 2.1 Front-Matter 解析器存在内容截断风险

**涉及文件：** `blog.js:136-153`

**问题描述：** Markdown 文章的 front-matter 解析使用 `md.indexOf('---', 3)` 查找结束标记。如果文章正文中包含 `---`（Markdown 水平分割线），且该分割线恰好位于 front-matter 结束标记之前被匹配到，会导致解析错误。例如 `hello-world.md` 第 37 行就是一个 `---` 水平分割线。

**当前代码：**
```javascript
if (md.startsWith('---')) {
  const end = md.indexOf('---', 3);  // ⚠️ 可能误匹配正文中的 ---
  if (end > 0) {
    const fm = md.slice(3, end).trim();
    content = md.slice(end + 3).trim();
    // ...
  }
}
```

**实际测试：** 对于 `hello-world.md`，由于 front-matter 的结束 `---` 出现在正文水平线 `---` 之前，当前代码恰好能正确工作。但这依赖于数据巧合，存在隐患。

**修复建议：**
```javascript
// 使用更严格的 front-matter 匹配（要求 --- 独占一行）
if (md.startsWith('---\n') || md.startsWith('---\r\n')) {
  const endMatch = md.slice(3).match(/\n---\r?\n/);
  if (endMatch) {
    const endIndex = 3 + endMatch.index;
    const fm = md.slice(3, endIndex).trim();
    content = md.slice(endIndex + endMatch[0].length).trim();
    // ...
  }
}
```

### 🟡 中风险

#### 2.2 博客照片视图存在加载闪烁

**涉及文件：** `blog.html:80-158`

**问题描述：** 照片博客视图的渲染流程如下：
1. 同步代码先以空的 `PHOTO_POSTS` 数组调用 `renderCards()` → 显示"暂无内容"
2. 异步加载 `posts/manifest.json` → 数据到达后重新渲染

这导致用户首次访问时会看到短暂的空白状态闪烁。

**修复建议：**
```javascript
// 方案一：添加加载状态指示器
const grid = document.getElementById('photoGrid');
grid.innerHTML = '<div class="empty-state">加载中...</div>';

// 方案二：在 HTML 中预置初始数据（如最近 3 篇），避免完全空白
```

#### 2.3 分享页搜索会隐藏占位卡片

**涉及文件：** `shares.html:119-131`

**问题描述：** 当分享列表不足 6 条时会添加"更多分享即将到来"的占位卡片（ghost card）。但搜索过滤逻辑遍历 `#shareGrid .sq-card` 时也会匹配并可能隐藏该占位卡片，导致搜索不匹配时页面完全空白。

**修复建议：**
```javascript
input.addEventListener('input', () => {
  const q = input.value.trim().toLowerCase();
  const cards = document.querySelectorAll('#shareGrid .sq-card:not(.ghost)');
  const ghost = document.querySelector('#shareGrid .sq-card.ghost');
  cards.forEach(card => {
    const text = card.textContent.toLowerCase();
    card.style.display = (q === '' || text.indexOf(q) !== -1) ? '' : 'none';
  });
  // 搜索时隐藏占位卡片，搜索清空时恢复
  if (ghost) ghost.style.display = q ? 'none' : '';
});
```

#### 2.5 博客标签过滤器与实际数据不匹配

**涉及文件：** `blog.html:29-35`

**问题描述：** 标签过滤栏硬编码了 5 个标签：`全部`、`日常`、`摄影`、`打卡`、`旅行`、`美食`。

```javascript
const tagSet = new Set();
posts.forEach(p => (p.tags || []).forEach(t => tagSet.add(t)));

// 方案二：至少保持硬编码标签与实际数据同步
```

#### 3.1 缩进风格不一致

```ini
# .editorconfig
```

#### 3.2 内联样式与外部 CSS 混用

**涉及文件：** `about.html:41-42`、`works.html:54`

**问题描述：** 项目中 JavaScript 存在三种组织方式：
