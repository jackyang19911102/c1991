现代 CSS 布局最佳实践 2026

Grid、Flexbox、Container Queries 的实战总结 — 从响应式到自适应

2026.06.20 · 作者分享

Grid、Flexbox、Container Queries 的实战总结。从响应式到自适应，掌握现代 CSS 布局的全部武器。

一、Flexbox — 一维布局的基石

Flexbox 适合处理单行或单列的布局场景。导航栏、卡片行、按钮组、表单项排列——这些都是一维布局的典型应用。掌握 justify-content、align-items、gap 和 flex 简写属性就足以应对大部分场景。

核心技巧

01
gap 代替 margin

使用 gap 属性设置子元素间距，比 margin 更清晰、更易维护。

02
flex: 1 实现等分

给所有子元素设置 flex: 1 即可实现等宽分布，无需计算百分比。

03
margin-auto 居中技巧

在 Flex 容器中，对单个子元素使用 margin: auto 可以实现完美的空间分配。

二、Grid — 二维布局的利器

当布局同时涉及行和列时，CSS Grid 是不二之选。卡片网格、页面整体布局、仪表盘面板——这些二维场景用 Grid 处理最为优雅。

常用模式

/* 自适应卡片网格 */
.card-grid {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(280px, 1fr));
  gap: 20px;
}

/* 经典两栏布局 */
.page-layout {
  display: grid;
  grid-template-columns: 1fr 320px;
  gap: 32px;
}

三、Container Queries — 真正的组件级响应式

Container Queries 让组件能够根据自身容器的尺寸而非视口尺寸来调整样式。这意味着同一个组件在不同宽度的父容器中可以呈现不同的布局，真正实现了可复用的响应式组件。

/* 定义容器 */
.card-wrapper {
  container-type: inline-size;
}

/* 根据容器宽度调整 */
@container (min-width: 400px) {
  .card {
    display: grid;
    grid-template-columns: 200px 1fr;
  }
}

四、实战建议

选择布局方案时遵循一个简单原则：一维用 Flexbox，二维用 Grid，组件级自适应用 Container Queries。三者配合使用而非互相替代，才能发挥各自的最大优势。
