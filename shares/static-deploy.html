静态网站部署指南

使用 GitHub Pages + GitHub Actions 实现零成本、自动化的静态网站部署流程

2026.06.15 · 作者分享

使用 GitHub Pages + GitHub Actions 实现零成本、自动化的静态网站部署流程。

一、GitHub Pages 设置

在仓库 Settings → Pages 中将部署源设为 GitHub Actions。这比直接从分支部署更灵活，允许在部署前进行构建、检查等自定义操作。

📋 前置条件

需要一个 GitHub 仓库（公开仓库免费使用 GitHub Pages，私有仓库需要付费）。推荐使用自定义域名以获得更好的品牌形象。

二、GitHub Actions 工作流

创建 .github/workflows/deploy.yml 文件，配置自动部署流程：

name: Deploy to GitHub Pages

on:
  push:
    branches: [master]

jobs:
  deploy:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      pages: write
      id-token: write

    steps:
      - uses: actions/checkout@v4

      - name: Setup Pages
        uses: actions/configure-pages@v4

      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: '.'

      - name: Deploy to GitHub Pages
        uses: actions/deploy-pages@v4

每次推送代码到 master 分支时，GitHub Actions 会自动运行此工作流，将静态文件部署到 GitHub Pages。

三、自定义域名

在仓库根目录添加 CNAME 文件，内容为你的域名（如 c1991.cn）。然后在 DNS 服务商处添加 CNAME 记录指向 username.github.io。

DNS 配置示例

类型: CNAME
主机记录: @ (或 www)
记录值: username.github.io
TTL: 600

四、总结

GitHub Pages + GitHub Actions 组合是个人博客和静态网站的理想部署方案：零成本、全自动、无需管理服务器。只需推送代码，一切自动完成。
